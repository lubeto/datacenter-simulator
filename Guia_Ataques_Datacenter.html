Ataques de Red

5 ataques
🌊
DDoS / SYN Flood
Critico Red
Que es

Inunda el servidor con millones de paquetes SYN (inicio de handshake TCP) sin completar la conexion, agotando los recursos de red y CPU. El servidor queda esperando respuestas que nunca llegan (half-open connections), bloqueando trafico legitimo.

Como funciona

El atacante (o botnet) envia masivamente paquetes SYN. El servidor responde SYN-ACK y espera el ACK final que nunca llega. Miles de conexiones half-open llenan la tabla de conexiones hasta saturation completa.

Metricas afectadas
Net IN ↑↑↑ Conexiones ↑↑↑ CPU ↑↑ Latencia ↑↑ RAM ↑
Como detectarlo en el simulador
1Net IN supera 800 MB/s en el dashboard principal
2Conexiones activas >5000 provenientes de multiples IPs
Mitigacion
1Rate limiting: limitar paquetes SYN por IP/segundo con iptables o nftables
2SYN Cookies: habilitar con sysctl -w net.ipv4.tcp_syncookies=1
3CDN / Scrubbing Center: absorber trafico malicioso antes del servidor
4Null-route temporal a rangos IP de origen del ataque
🔑
Brute Force Attack
Alto Red
Que es

Ataque que intenta adivinar credenciales probando combinaciones de usuario y contrasena de forma automatizada y repetitiva contra servicios de autenticacion (SSH, RDP, HTTP, FTP).

Como funciona

Herramientas como Hydra o Medusa envian cientos de intentos de login por segundo usando diccionarios de contrasenas comunes, credenciales de brechas previas (credential stuffing) o patrones generados automaticamente.

Metricas afectadas
Conexiones ↑↑ CPU ↑ Net IN ↑ RAM —
Como detectarlo en el simulador
1Conexiones elevadas desde pocas IPs con patron repetitivo
2Logs de autenticacion con cientos de intentos fallidos por minuto
Mitigacion
1fail2ban: bloqueo automatico tras N intentos fallidos configurables
2MFA: segundo factor de autenticacion obligatorio en todos los servicios
3Bloqueo por IP: ban temporal o permanente segun umbral
4Contrasenas largas (>16 caracteres) gestionadas con un password manager
🔍
Port Scan
Reconocimiento Red
Que es

Tecnica de reconocimiento que mapea puertos abiertos y servicios disponibles en un host. Permite al atacante identificar la superficie de ataque antes de lanzar un exploit especifico.

Como funciona

Herramientas como nmap envian sondas a rangos de puertos (1-65535). El SYN Scan (-sS) es sigiloso porque no completa el handshake. El Version Scan (-sV) identifica versiones exactas de servicios para buscar CVEs conocidos.

Metricas afectadas
Conexiones ↑↑ Net IN ↑ CPU — RAM —
Como detectarlo en el simulador
1Conexiones elevadas desde pocas IPs hacia muchos puertos distintos
2Patron de trafico de sondeo sin transferencia de datos real
Mitigacion
1IDS/IPS: Snort o Suricata con reglas de deteccion de scan activo
2Firewall default-deny: solo puertos estrictamente necesarios expuestos
3Port knocking: requiere secuencia de puertos como llave de acceso
4Honeypots: detectar y registrar reconocimiento activo en tiempo real
🕵️
ARP Spoofing / MITM
Critico Red
Que es

El atacante envia respuestas ARP falsas para asociar su direccion MAC a la IP de otro host (gateway o servidor), redirigiendo todo el trafico de la victima a traves de su maquina sin que ella lo note.

Como funciona

ARP no tiene autenticacion. El atacante envia ARP Replies no solicitados con IP del gateway = su MAC. La victima actualiza su tabla ARP y envía todo el trafico al atacante, quien lo reenvía al destino real (transparent proxy). Herramientas: arpspoof, bettercap.

Metricas afectadas
Net IN/OUT ↑↑ Latencia ↑↑ Conexiones ↑ CPU ↑
Como detectarlo en el simulador
1Trafico de red anormalmente duplicado con latencia elevada sostenida
2Retransmisiones TCP frecuentes sin sobrecarga aparente de CPU
Mitigacion
1Dynamic ARP Inspection (DAI) habilitado en switches capa 2
2802.1X: autenticacion de dispositivos antes de acceder a la red
3ARP estatico en infraestructura critica (gateway, servidores clave)
4Segmentacion con VLANs para aislar segmentos sensibles

Ataques de Hardware

2 ataques
🧠
Memory Leak
Alto Hardware
Que es

Falla donde un proceso reserva memoria RAM pero nunca la libera, acumulandola progresivamente hasta agotar los recursos del sistema. Puede ser una falla de software o explotada deliberadamente para causar denegacion de servicio.

Como funciona

El proceso aloca memoria (malloc, new) sin liberarla (free, delete) en ciclos repetitivos. La RAM usada crece continuamente sin estabilizarse. Cuando la RAM se agota, el sistema activa swap en disco y eventualmente el kernel mata el proceso (OOM Killer).

Metricas afectadas
RAM ↑↑↑ CPU ↑ Net IN — Disk I/O —
Como detectarlo en el simulador
1RAM crece de forma continua y sostenida sin estabilizarse nunca
2CPU se mantiene relativamente normal mientras RAM escala
Mitigacion
1Monitorear tendencia de RAM con alertas proactivas al superar 85%
2Reiniciar el proceso afectado como workaround temporal inmediato
3Aplicar patch del software o actualizar a version corregida
4Limitar memoria maxima del proceso con cgroups (memory.limit_in_bytes)
💿
Disk Failure / RAID Failure
Critico Hardware
Que es

Falla fisica o logica de uno o mas discos del array de almacenamiento. En entornos RAID, la falla de un disco degrada el array y expone los datos ante una segunda falla. El Disk I/O se vuelve extremadamente alto por los reintentos del sistema.

Como funciona

Los discos mecanicos (HDD) fallan por desgaste de cabezales o bad sectors. Los SSDs por ciclos de escritura agotados. Cuando el controlador RAID detecta la falla, marca el disco como degradado y activa el modo rebuild al insertar un disco nuevo, operacion costosa en I/O.

Metricas afectadas
Disk I/O ↑↑↑ Disk Used ↑↑ CPU ↑ RAM ↑
Como detectarlo en el simulador
1Disk I/O supera 150 MB/s con errores constantes en los logs
2Disk Used creciendo anomalamente junto con alertas SMART
Mitigacion
1Monitorear SMART con smartctl -a /dev/sdX y alertas automaticas
2Reemplazar el disco fallido de inmediato para no perder redundancia
3Rebuild RAID: mdadm --manage /dev/md0 --add /dev/sdX
4Backups 3-2-1: 3 copias, en 2 medios distintos, 1 offsite o nube

Seguridad Fisica — SST

2 eventos
🌡️
Falla Termica
Critico SST Fisico
Que es

La temperatura en el rack supera umbrales seguros (>27 C ambiente, >80 C en componentes), poniendo en riesgo hardware y disponibilidad. El hardware activa throttling o apagado de emergencia por proteccion termica del UEFI/BIOS.

Como funciona

Cuando el sistema CRAC/CRAH falla o el flujo de aire se bloquea, el calor generado por los equipos se acumula. La temperatura sube progresivamente. CPUs activan throttling al llegar al TjMax. Si persiste, el firmware apaga el servidor para prevenir dano permanente.

Sensores SST activados
🌡️ Temperatura ALERTA ⚡ Energia ALERTA 💨 Humo Normal 🚪 Acceso Normal
Como detectarlo en el simulador
1Panel SST muestra sensor de temperatura en estado ALERTA rojo
2CPU throttling visible: el porcentaje cae abruptamente por proteccion
Mitigacion
1Verificar estado de unidades CRAC/CRAH del datacenter de inmediato
2Hot-aisle / Cold-aisle containment para separar flujos de aire
3Blanking panels en espacios vacios del rack para guiar el aire frio
4Protocolo de apagado ordenado si la temperatura supera 30 C
🚨
Acceso No Autorizado
Critico SST Fisico
Que es

Entrada fisica al datacenter por personal no autorizado, visitante sin escolta o mediante tecnicas de ingenieria social. Permite al intruso interactuar directamente con hardware critico, robar equipos o insertar dispositivos maliciosos.

Como funciona

El vector mas comun es el tailgating: el atacante sigue a una persona autorizada al cruzar puertas de acceso controlado. Tambien puede usar credenciales clonadas (RFID/NFC), suplantar a un tecnico de mantenimiento o sobornear al personal de seguridad fisica.

Sensores SST activados
🚪 Control Acceso ALERTA 📹 CCTV Activo 🌡️ Temperatura Normal 💨 Humo Normal
Como detectarlo en el simulador
1Panel SST muestra sensor de control de acceso en estado ALERTA
2Registro de acceso muestra entrada sin credencial valida registrada
Mitigacion
1Bloquear inmediatamente la credencial comprometida en el sistema
2Revisar logs de acceso en el SIEM para correlacionar el incidente
3Doble factor fisico obligatorio: PIN + badge RFID + biometria
4Mantraps / airlocks en el acceso principal para evitar tailgating

Ataques SSL/TLS

2 ataques
🔓
SSL/TLS Downgrade Attack
Critico SSL/TLS
Que es

El atacante manipula la negociacion TLS para forzar al servidor a usar una version antigua (TLS 1.0, TLS 1.1, SSL 3.0) que contiene vulnerabilidades conocidas y explotables como POODLE, BEAST o FREAK.

Como funciona

El atacante actua como MITM e intercepta el Client Hello. Modifica la lista de cipher suites para incluir solo protocolos debiles. El servidor acepta el protocolo antiguo. Con TLS 1.0 o SSL 3.0 activo, el atacante puede usar POODLE/BEAST para descifrar el trafico capturado.

Metricas afectadas
🔒 Alerta SSL: TLS 1.0/1.1 Latencia ↑↑ Net OUT ↑ CPU —
Como detectarlo en el simulador
1Panel SSL muestra alerta: version negociada = TLS 1.0 o TLS 1.1
2Latencia elevada sugiere proxy MITM insertado en la ruta
Mitigacion
1Configurar nginx/Apache: ssl_protocols TLSv1.2 TLSv1.3;
2Deshabilitar explicitamente SSL 3.0, TLS 1.0 y TLS 1.1
3HSTS: header Strict-Transport-Security con max-age largo
4Validar configuracion con SSL Labs: ssllabs.com/ssltest (objetivo: A+)
📋
Certificado SSL Expirado
Alto SSL/TLS
Que es

Un certificado X.509 tiene fecha de vencimiento. Al expirar, navegadores y clientes API rechazan la conexion HTTPS con error de seguridad. Aunque el servicio este corriendo correctamente, es inaccesible para los usuarios.

Como funciona

El certificado incluye los campos notBefore y notAfter. Al pasar la fecha notAfter, el cliente verifica que el cert sea invalido y termina la conexion con error SSL_ERROR_EXPIRED_CERT. APIs con mutual TLS y microservicios tambien fallan en cascada.

Metricas afectadas
🔒 Cert Expirado: dias = 0 HTTPS DOWN Net IN ↓↓ CPU —
Como detectarlo en el simulador
1Panel SSL muestra alerta: certificado expirado con 0 dias restantes
2Net IN cae abruptamente: clientes rechazan la conexion inmediatamente
Mitigacion
1Renovar inmediatamente: certbot renew (Let's Encrypt — gratuito)
2Automatizar renovacion con cron: 0 3 * * * certbot renew
3Alertas preventivas a 30, 14, 7 y 1 dia antes del vencimiento
4Certificate Manager en AWS/GCP/Azure para renovacion automatica

Como Funciona el DC Monitoring Simulator

Flujo de una sesion de practica — 4 pasos del aprendiz al reporte

1

Instructor activa ataque

El instructor selecciona el tipo de ataque desde el panel de control del simulador. El aprendiz no sabe cual es — debe identificarlo por las metricas.

2

Aprendiz observa metricas

Analiza en tiempo real: CPU %, RAM %, Net IN (MB/s), Disk I/O (MB/s), Conexiones activas. Identifica cual metrica esta fuera de rango normal.

3

Diagnostico Guiado

Usa la herramienta de diagnostico del simulador para correlacionar las senales observadas, clasificar el tipo de ataque y validar el patron de metricas.

4

Bitacora y Reporte

Documenta el incidente: tipo de ataque identificado, metricas observadas con valores, acciones de mitigacion aplicadas y tiempo total de respuesta.
